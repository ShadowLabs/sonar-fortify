ABSTRACT

The identified function sometimes fails to release a system resource.

EXPLANATION

The program can potentially fail to release a system resource.


In this case, there are program paths on which the resource allocated in at line is not released.



Resource leaks have at least two common causes:

- Error conditions and other exceptional circumstances.

- Confusion over which part of the program is responsible for releasing the resource.

Most unreleased resource issues result in general software reliability problems, but if an attacker can intentionally trigger a resource leak, the attacker might be able to launch a denial of service attack by depleting the resource pool.

Example 1: The following method never closes the stream that it reads from. 


...

CFIndex numBytes;

do {

    UInt8 buf[bufferSize]; 

    numBytes = CFReadStreamRead(readStream, buf, sizeof(buf));

    if( numBytes > 0 ) {

        handleBytes(buf, numBytes);

    } else if( numBytes < 0 ) {

        CFStreamError error = CFReadStreamGetError(readStream);

        reportError(error);

    }

} while( numBytes > 0 );
