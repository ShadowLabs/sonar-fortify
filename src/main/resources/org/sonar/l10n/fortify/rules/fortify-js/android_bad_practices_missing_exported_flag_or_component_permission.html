ABSTRACT

The program does not explicitly assign an access permission to a component.

EXPLANATION

Any application can access public components that are not explicitly assigned an access permission in their manifest definition. Android content provider are exported by default for applications that set either android:minSdkVersion or android:targetSdkVersion to "16" or lower. For applications that set either of these attributes to "17" or higher, the default is "false".

Example 1: Below is an example of an Android content provider declared without an explicit access permission or exported flag. 

 <provider android:name=".ContentProvider"/>
