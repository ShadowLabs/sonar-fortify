ABSTRACT

The identified call uses methods which follow symbolic links.

EXPLANATION

Certain identified functions are known to blindly follow symbolic links. When this happens, your application will open, read, or write data to the file that the symbolic link points to instead of the representation of the symbolic link. An attacker can fool the application into writing to alternate or critical system files or provide compromised data to the application.

Example 1: The following code utilizes functions which follow symbolic links: 


...

	struct stat output;

	int ret = stat(aFilePath, &output);

	// error handling omitted for this example

	struct timespec accessTime = output.st_atime;

...
